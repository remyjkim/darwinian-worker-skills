# worker-bound-mind

A project selects at most one Worker. Cards in that Worker's locked Blueprint
closure compose its persona, beliefs, capabilities, and optional Mind seeds.
There is no separate active-Mind stack and no hidden machine inheritance.

Project Worker selection is explicit through `drwn use`; changing that
selection is an Operator workflow. Provisioning or operating the selected
Worker's live Mind is a separate `drwn worker mind` workflow.
